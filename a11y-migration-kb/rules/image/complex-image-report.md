# 必須ルール
画像内に文字（バナー・ポスター・チラシの見出しや告知文、日付、案内文など）がある場合、その文字も画像の内容なので、可能な範囲で画像名に含める。そのうえで、画像の内容（画像内の文字を含む）を画像名一行＝100文字程度に収められるかで扱いを分ける。

- **収まる場合**（例：小さな告知バナー）は、内容（画像内の文字を含む）をそのまま画像名にする。「詳細は以下」は付けない（例：「7月10日は休館日です」）。
- **収まらない複雑な画像**（グラフ・楽譜・チラシ・凡例・ポスター等で内容が多い）は、本文に内容説明テキストを追記するか解説ページへのリンクを提供し、画像名の末尾に「詳細は以下」を付す。「詳細は以下」の前の主題部分は、分類語一語（「グラフ」「ポスター」等）で終わらせず、alt-text.md の①主題＋②様子＋③+α情報の考え方でできるだけ具体的に書く（例：「絶景スポットとご当地グルメを紹介する観光地特集のポスター」）。グラフの全数値・チラシの全項目・画像内の全文字のような網羅的な詳細は画像名に詰め込まず、本文への追記または報告欄への起票で扱う。

長さは miChecker の上限（150文字超は「長すぎ」）未満に収める（目安100文字程度）。複雑な画像の場合は、あわせて自治体向けコメントを報告欄に起票する。

# 例
## ケース1: 100文字に収まる告知バナー（文字をそのまま入れ、接尾辞は付けない）
```before
バナー画像
```
```after
7月10日は休館日です
```
ポイント: 画像内の文字を代替テキスト一行に収められる場合は、その文字をそのまま画像名にする。「詳細は以下」は不要。

## ケース2: 集計グラフ（主題を具体的に残す）
```before
グラフ
```
```after
人口推移の集計結果を示す折れ線グラフ 詳細は以下
```
ポイント: 「グラフ」の一語で終わらせず、何のグラフか（主題）を具体的に書く。全数値は本文へ追記し、報告欄に「グラフの内容説明を画像に続けてテキスト掲載してください」と起票する。

## ケース3: 観光特集のポスター（主題を具体的に残す）
```before
観光地特集のポスター
```
```after
絶景スポットとご当地グルメを紹介する観光地特集のポスター 詳細は以下
```
ポイント: 「観光地特集のポスター」だけでは内容が伝わりにくい。ポスターが何を紹介しているか（主題）を具体的に書く。掲載されている個々の写真や店名などの網羅的な内容は、本文への追記または報告欄への起票で扱う。

## ケース4: alt属性が長すぎる（150文字超）
```before
<img src="chart.png" alt="令和3年度から令和8年度までの人口推移を示す折れ線グラフで、令和3年は12万3千人、令和4年は12万1千人、令和5年は11万9千人、令和6年は11万7千人、令和7年は11万5千人、令和8年は11万3千人と年々減少傾向にあることを示している（以下略、150文字超）">
```
```after
<img src="chart.png" alt="令和3〜8年度の人口推移を示す折れ線グラフ 詳細は以下">
（詳しい数値は本文またはaria-describedbyで参照できる別要素に記載する）
```
ポイント: 全数値を詰め込むと読み上げの負担が大きい。主題（何を示すグラフか）は具体的に残しつつ、個々の数値は本文側（aria-describedby等）へ分離する。
