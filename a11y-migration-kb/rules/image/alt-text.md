# 必須ルール
画像に内容がある場合、その内容をテキストで説明する代替テキスト（画像名）を設定する。分類語（「写真」「画像」等）だけでは不十分。「電話で『こんな画像だよ』と言葉で伝える」イメージで、下記の①主題＋②様子＋③+α情報を一文に整えて画像名を作る。移行段階で文字起こしは不要だが、適切な画像名へ修正する。画像内容が近接するテキスト（キャプション・本文）で十分説明されていれば画像名の省略も可。長さは内容が伝わる範囲で最大100文字程度まで（miCheckerは150文字超を「長すぎ」と指摘するため、それ未満に収める。alt一行で説明しきれない複雑な画像は complex-image-report.md で扱う）。

## 画像名の考え方（①主題＋②様子＋③+α情報を一文に）
- **① 主題（what／何が）**: 画像の主役・主語にあたるもの。例）桜の木が、3階建ての白い建物が、作業着を着た男性が、ランドセルを背負った小学生たちが
- **② 様子（how／どうなっている・何をしている・どういう状態）**: 述語にあたるもの。例）満開に咲いている、建っている、談笑している、壇上で話している、画用紙に絵を描いている
- **③ +α情報**: ①②以外で画像から分かる情報を補う。観点は「どこで（建物の傍に／広い道の端で）」「何色（服の色・花の色など）」「誰と（ゆるキャラと／市長と／子どもたちと）」「画像の主役以外の人・ものの様子」など。例）道路の脇で、白い建物の近くで、スクリーンを指さしながら、緑色のハッピを着た、傍に犬がいる
- 上記①②③を一文に収まるように整えれば画像名の完成。語順は自然に読める順でよく、末尾は「〜の写真」等で結んでよい。
- 並べた後は一度読み返し、言い回しに違和感があれば言葉を並べ替える。例）「多くの桜が咲いていて、遠くに山脈が見える写真」→「山脈を背景に咲き誇る、多くの桜の写真」。

## 移行元の画像名は信用しない
移行元ページの画像名は「イベント写真」「授与式」「消防訓練1」「jisseki」「（イラスト）ふっけいくん」のように、機械的・便宜的に付けられていることがほとんどで、そのまま取り込まれているだけ。元の画像名が何であれ、上記①②③の基準で必ず付け直す。画像名が未設定で「●●●●…」等が入っている場合も同様に修正する。

## 省略できるのは「十分に説明されている」場合だけ
近接するテキスト（キャプション・本文）で画像の内容が十分に説明されていれば画像名は省略可。ただし説明が「日時」「場所」だけ等、画像の内容として不十分な場合は省略できず、画像名が必要。省略してよいか判断に迷う場合は自己判断せず質問する。

## 判断できない画像は報告する（エスカレーション）
ページ内の文脈からも適切な画像名を判断できない場合は、無理に付けず報告欄へ「上から（横並びの場合はさらに左から）何枚目の画像か」を明記して報告し、自治体（顧客）側での確認を依頼する。例）「2枚目と5枚目の画像は、こちらで何の画像か判断がつかなかったため、○○市様でも画像名の確認をお願いします。」。なお、内容が完全には分からなくても被写体の種類が分かる場合は、分かる範囲で画像名を付ける（例：「機械の部品の写真」「大きな機械の写真」）。

# 例
## ケース1: 桜（①桜の木が ＋ ②満開に咲いている ＋ ③道路の脇で）
```before
公園の写真
```
```after
道路の脇で桜の木が満開に咲いている写真
```

## ケース2: 男性（①作業着を着た男性が ＋ ②笑っている ＋ ③ミカンを持ちながら）
```before
男性の写真
```
```after
ミカンを持って笑っている、作業着を着た男性の写真
```

## ケース3: 市長（①市長が ＋ ②壇上で話している ＋ ③マイクを持って、スクリーンを指さしながら）
```before
市長の写真
```
```after
マイクを持ち、スクリーンを指さしながら、壇上で話している市長の写真
```

## ケース4: サツキ（①サツキの花が ＋ ②満開に咲いている ＋ ③ピンク色の）
```before
サツキの写真
```
```after
満開になっているピンク色のサツキの花の写真
```

## ケース5: 画像マップ(area要素)のalt属性
```before
<map name="siteMap"><area shape="rect" coords="0,0,100,50" href="/kosodate.html" alt="画像"></map>
```
```after
<map name="siteMap"><area shape="rect" coords="0,0,100,50" href="/kosodate.html" alt="子育て支援ページへ"></map>
```
ポイント: 画像マップのarea要素は「内容」ではなく「リンク先」が分かる代替テキストにする（①②③の被写体説明ではなく、飛び先を示す）。
