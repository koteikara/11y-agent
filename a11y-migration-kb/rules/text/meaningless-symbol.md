# 必須ルール
音声読み上げで意味が伝わらない記号は、そのまま残さない。

- **矢印記号（→ ↓ ↑ ← ⇔ ⇒）**: 読み上げソフトでは読み飛ばされるため、方向・流れを示す矢印は文脈に応じた言葉に置き換える。「AからB」の意味なら「から」、「AへB」なら「へ」など、文意に合わせて選ぶ（機械的に「から」へ固定しない）。判断が難しい場合や、置き換えでは不自然な場合は、レイアウトの見直しを検討するか質問する。
- **装飾記号（○ ● ◎ □ ■ ◇ ◆ ☆ ★）**: 装飾目的だけの記号は削除する（必要に応じて半角スペースに置き換える）。ただし、箇条書きの代わりに使われている場合はリスト（list.md）に、見出しの代わりに使われている場合は見出し（heading-required.md）に変換する。固有名詞の一部（例：「わくわく☆イベント」）はそのまま残す。
- **注釈の「※」**: note-symbol.md で扱う。

# 例
## ケース1: 方向を示す矢印
```before
受付（1階）→ 相談室（2階）
```
```after
受付（1階）から相談室（2階）へ
```
ポイント: この例では「から」「へ」が文意に合う。別の文脈では「〜の次に」「〜になる」等が適切なこともあるため、文意に応じて言葉を選ぶ。

## ケース2: 装飾目的の記号
```before
★★イベントのお知らせ★★
```
```after
イベントのお知らせ
```
ポイント: 装飾の「★」は削除する。この行が見出しの役割なら見出しに、箇条書きの見出し行ならその旨で整える。「わくわく☆イベント」のように固有名詞の一部であればそのまま残す。
