# GOAL2 作業者向け操作マニュアル

CMSへ登録する本文HTMLのアクセシビリティ上の問題を確認し、修正候補を1件ずつ「採用」「文言を調整」「却下」「要確認」のいずれかに判断していくための画面です。この文書は、実際に画面を操作する作業者向けに、開発の専門知識なしで読める形でまとめています。

## この画面でできること

1. CMSへ登録する予定の本文HTML断片を貼り付ける
2. アクセシビリティ上の問題点(画像の代替テキスト不足、リンク文言が曖昧、表の構造など)の修正候補を自動生成する
3. 候補を1件ずつ確認し、採用するか・文言を直して採用するか・却下するか・要確認にするかを判断する
4. すべて判断し終えたら、CMSに貼り付ける最終HTMLと、判断内容の記録(証跡)を出力する

## 全体の流れ(4ステップ)

```
① HTML入力  →  ② 候補生成  →  ③ 候補レビュー  →  ④ 出力
```

画面左下に出る**「次にやること」パネル**が、今どのステップにいるか・次に何をすればよいかを常に教えてくれます。迷ったら、まずこのパネルに従ってください。パネルは移動でき(パネル右上の矢印アイコン→矢印キーで移動)、右上の×で一時的に閉じられます。

## ステップ① HTML入力

画面上部の入力欄に、以下を入力します。

| 項目 | 内容 |
|---|---|
| ページ名 | CMS登録後のページタイトル |
| 旧URL | 移行元(旧サイト)のページURL。画像のAI解析などで使われることがあります |
| CMS登録先 | どのメニュー・カテゴリに登録するページかのメモ |
| 作業者 | 自分の名前・ID |
| 本文HTML | CMSへ登録する本文のHTML断片を貼り付ける欄 |

**修正基準**の切り替え(入力欄の下)では、次のどちらで候補を出すかを選べます。

- **KB全ルール（miChecker含む）**: ナレッジベースの全ルールで判定(通常はこちら)
- **miChecker指摘対応のみ**: miChecker(公的な検査ツール)が指摘する項目に絞って確認したい場合

**サンプル投入**は、練習・動作確認用のダミーデータを入力欄に入れる機能です。**実際の案件データではないので、本番作業では使いません。**

入力が終わったら「**候補生成**」ボタンを押します。

## ステップ② 候補生成

候補生成中はボタンが「生成中」または「**AIで確認中**」と表示されます。

- 「生成中」: 通常のルール判定(数秒で終わります)
- 「AIで確認中」: 一部の候補についてAI(Gemini)による内容確認が行われている状態です。ページの内容によっては数十秒かかることがあります。この間は画面全体に半透明のオーバーレイが表示され、他の操作(サンプル切替えなど)もできなくなります。**表示が消えるまで待ってください。**

候補が生成されると、左側に「修正候補」の一覧が表示されます。AIを使った確認が行われた候補には、画面上に「(AI判定)」「(AI提案)」のような注記が付きます。この場合も内容は必ず人間が確認してから判断してください。

## ステップ③ 候補レビュー(メインの作業)

### 候補一覧カードの色の意味

候補一覧の各カードは、背景色で今の状態がひと目でわかるようになっています。

| 背景色 | 状態 |
|---|---|
| 白 | 未処理(まだ判断していない) |
| 青 | 今選んで詳細を見ている候補(選択中) |
| 濃いグレー | 判断済み(採用・文言を調整して採用・却下・要確認のいずれか) |
| 薄いグレー | 同じ箇所の別の候補が処理されたことで、自動的に対応不要になった候補 |

カード左端の縦線の色は、判断の種類(緑=採用系、赤=却下、オレンジ=要確認)を表します。

複数の候補が青枠でひとまとまりに囲まれている場合、それらは**同じ箇所に対する複数の直し方の選択肢**です(「同じ箇所の代替手段」)。詳細パネルでどれか1つを選んで判断します。

### 候補を選んで詳細を確認する

一覧からカードをクリックすると、右側(中央)の詳細パネルに以下が表示されます。

- **この候補で変わること**: 何がどう変わるかの要約
  - この下にある「❓ このルールについて詳しく確認する」を押すと、このルールが何のためにあるのか(WCAG/JIS番号・元になっているルール文・実際の修正前後の例)を、画面中央のポップアップで集中して確認できます。判断に迷ったとき・このルールの意味が気になったときに開いて学習できる場所です。「閉じる」ボタン・背景クリック・Escキーのいずれかで閉じられます。
- **修正方法**: 直し方が複数ある場合、ここで選択します(「おすすめ」が先頭に表示されます)
- **見た目の比較**: 修正前・修正後を実際の見た目で比較できます(HTMLを読めなくても違いがわかります)
- **各種情報**(「詳細を見る」を開くと表示): 分類、状態、確度、CMSへの反映方法、問題点、判断理由、根拠となるWCAG/JIS番号・KBの出典

いちばん右のプレビュー欄には、選択中の候補が本文HTML内のどこに当たるかがハイライト表示されます。欄が狭くて見づらいときは、右上の「拡大」ボタンを押すと画面ほぼ全体にプレビューを拡大表示できます。閉じるには「閉じる」ボタン、背景クリック、またはEscキーのいずれかを使います。

### 4つの判断ボタン

| ボタン | 意味 |
|---|---|
| 採用 | 提示された修正内容をそのまま採用する |
| 文言を調整 | 修正内容(文言やリンク先など)を自分で直してから採用する |
| 却下 | この修正は不要と判断する(理由を残せます) |
| 要確認 | 自分では判断できない・上長やSVの確認が必要 |

画像の代替テキスト候補には、判断ボタンの上に青枠の「AI画像名候補」欄が出ることがあります。この場合、内容を確認して「修正後HTMLへ投入」を押してから採用・調整する必要があります(採用ボタンが無効になっている場合はこの投入操作が未実施です)。

見出し(見出し階層の順守・見出しの追加など)の候補で「文言を調整」を押すと、見出しの文言だけでなく「見出しレベル」(h2〜h6)も選び直せます。AIやシステムが提案したレベルが実際のページ構造と合わないと感じたときは、ここで作業者自身が正しいレベルへ変更してから採用してください。この機能は、その箇所にすでに何らかの見出し候補が出ている場合にのみ使えます(候補が1件も出ていない見出しには使えません)。

修正前後を比べても見た目が同じ候補(画像の代替テキストや外国語判定など、読み上げにのみ関わる変更)は、「見た目の比較」のカードにマウスを重ねると、実際に何が変わったかがツールチップで表示されます。

判断に迷ったときは、無理に採用・却下を選ばず**「要確認」**にしておくのが安全です。

### まとめて処理する(一括採用)

同じような対応でよい候補が複数ある場合、一覧のチェックボックスで複数選択し、「**チェックを一括採用**」でまとめて採用できます。ただし内容を確認せずに一括採用すると誤りに気づきにくいため、内容に自信がない候補は個別に確認することを推奨します。

### 次の候補へ進む

詳細パネル右上の「**前**」「**次の未処理**」ボタンで、候補間を移動できます。「次の未処理」は、まだ判断していない候補まで自動的に飛びます。

## ステップ④ 出力

画面下部の「出力」を開くと、次の3つが確認できます。

- **最終HTML**: すべての判断を反映した、CMSへ貼り付け用のHTML。「コピー」ボタンでコピーできます。
- **注意**: HTMLには反映されないが、CMS登録時に人が確認すべき事項の一覧(コピー対象ではありません)。
- **証跡**: 候補ごとの判断内容の記録。「JSONコピー」または「CSV」でダウンロードできます。作業記録として保存してください。

候補一覧の右上「完了可/未完了」の表示が「完了可」になっていれば、すべての候補が判断済みです。**未完了のまま出力しないよう注意してください。**

## よくある質問

**Q. 「AIで確認中」が長い、途中で止まったように見える**
ページの内容量やAI呼び出し件数によっては数十秒かかることがあります。表示が変わらず数分待っても終わらない場合は、担当者に連絡してください。

**Q. AIの提案は無条件で信じてよい?**
いいえ。AIによる候補には必ず「(AI判定)」「(AI判定の下書きです。内容を確認してください)」のような注記が付きます。内容が実態と合っているか必ず人間が確認してから採用してください。確度が低い、あるいは内容に疑問がある場合は「要確認」を選んでください。

**Q. サンプル投入したデータのまま作業してしまった**
サンプルは練習用のダミーデータです。本番のCMS登録には使わないでください。「リセット」で入力欄をクリアしてやり直せます。

**Q. 候補が1件も出ない**
本文HTMLに明らかな問題がない場合は候補が0件になることもあります。ただし「注意」欄に確認事項が出ている場合があるので、必ず出力欄を確認してください。

## 用語集

| 用語 | 意味 |
|---|---|
| 候補(修正候補) | 自動検出されたアクセシビリティ上の問題と、その直し方の提案 |
| KB(ナレッジベース) | この画面が参照している、アクセシビリティ対応ルールの一覧 |
| miChecker | 総務省が提供する公的なアクセシビリティ検査ツール。本ツールの一部の判定基準の参考にしている |
| WCAG/JIS | アクセシビリティの達成基準(国際基準/日本産業規格) |
| 証跡 | 誰が・いつ・どの候補を・どう判断したかの記録 |
